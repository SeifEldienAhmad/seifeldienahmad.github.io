**Contributing Guidelines**
Thank you for considering contributing to my personal website! This project is a space to showcase my work, but I welcome suggestions, bug reports, and improvements from the community.
**How to Contribute**
There are three main ways you can help improve this project:
  1. Reporting Bugs
  2. Suggesting Features or Improvements
  3. Submitting Code Changes (Pull Requests)
**1. Reporting Bugs**
If you find an error, a broken link, or unexpected behavior, please let me know!
Check Existing Issues: Before creating a new issue, please check the existing list of issues to see if the problem has already been reported.
Create a New Issue: If the bug is new, use the Bug Report template (if available) or include the following information:
Description: A clear and concise description of the bug.
Steps to Reproduce: The exact steps to follow to replicate the issue (e.g., "1. Go to '/about' page. 2. Click the 'Contact' button. 3. See error").
Expected Behavior: What you expected to happen.
Actual Behavior: What actually happened.
Environment: Your operating system and browser (e.g., Windows 11, Chrome 120).
**2. Suggesting Features or Improvements**
Got an idea for a better layout, a new feature (like a dark mode toggle), or a performance tweak?
Check Existing Issues: Search for similar suggestions first.
Create a New Issue: Use the Feature Request template (if available) or describe the suggestion clearly and concisely. Explain why you think this change would benefit the site.
**3. Submitting Code Changes (Pull Requests)**
I welcome code contributions! To ensure a smooth process, please follow these steps:
A. Setup and Workflow
Fork the Repository: Create a copy of the project in your own GitHub account.
Clone the Repository: Clone your fork to your local machine.
Create a New Branch: Always create a dedicated branch for your work. Use a descriptive name (e.g., fix/broken-footer-link or feature/add-dark-mode).
git checkout -b your-branch-name


Make Changes: Implement your bug fix or new feature.
Commit Changes: Write clear, concise commit messages that explain what you changed and why.
```git commit -m "feat: implement dark mode toggle using CSS variables"```


Push to Your Fork: Push your new branch to your forked repository.
```git push origin your-branch-name```


Open a Pull Request (PR): Open a PR from your branch back to the main branch of the original repository.
B. Pull Request Checklist
When opening your PR, please ensure you have completed the following:
[ ] Clear Title and Description: The title should summarize the change, and the description should explain the motivation for the change and any testing performed.
[ ] Clean Code: Your code follows existing styles and standards.
[ ] Single Focus: The PR should address only one bug or feature. If you have multiple changes, please submit separate PRs.
[ ] No Linting Errors: Ensure all linting checks pass locally before submission.
